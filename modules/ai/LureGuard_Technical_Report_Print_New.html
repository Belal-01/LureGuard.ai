التقرير التقني الشامل لمشروع LureGuard

وثيقة تعريفية وتشغيلية لشرح المشروع بشكل واضح لشخص لا يعرف عنه شيئًا: الهدف، طريقة العمل، المدخلات والمخرجات، وخطوات الاستخدام.

نوع المستند: Technical Report
اللغة: العربية
تنسيق الطباعة: A4

1) تعريف المشروع

مشروع LureGuard هو خدمة API أمنية تقوم بتحليل تنبيهات المصادقة (Authentication Alerts) القادمة من Wazuh، وتحويلها إلى ميزات رقمية سلوكية، ثم تمريرها إلى نموذج تعلم آلة لإخراج درجة خطورة وقرار نهائي.

2) ماذا يقدم المشروع عمليًا؟

  • استقبال الحدث الأمني بصيغة JSON.
  • حساب ميزات سلوكية زمنية (f1..f8) على نافذة زمنية محددة.
  • تطبيق نموذج تصنيف لإعطاء احتمالية المخاطر risk_score.
  • إرجاع قرار تشغيل واضح: allow أو block.
  • تطبيق whitelist لحماية الكيانات الموثوقة من الحظر الخاطئ.
  • مراقبة انحراف البيانات (Data Drift) لضمان استقرار الأداء.

3) المعمارية المختصرة

Wazuh Event JSON
    |
    v
FastAPI endpoint: POST /wazuh/event
    |
    v
Feature Extraction (f1..f8)
    |
    v
Scaler + RandomForest model
    |
    v
Decision Engine (threshold)
    |
    v
Output JSON + Drift Status

4) الملفات الأساسية داخل المشروع

الملف المسؤولية
main.py تشغيل API، تحميل model/scaler، تنفيذ القرار، وإرجاع الاستجابة.
LureGuardExtractor.py استخراج الميزات من السجلات على نافذة زمنية.
feature_contract.py تعريف رسمي للميزات وصيغها ووحداتها.
whitelist.py التحقق من whitelist عبر IP/CIDR/user/expiry.
drift_monitor.py قياس PSI لمراقبة الانحراف في الميزات الحساسة.
train_intrusion_model.py تدريب النموذج وإنتاج ملفات artifacts.
artifacts/ model.joblib, scaler.joblib, metrics.json, drift_baseline.json, whitelist.json

5) شرح الميزات (f1..f8)

الميزة المعنى الوحدة
f1 عدد المحاولات داخل النافذة الزمنية count
f2 نسبة المحاولات الفاشلة ratio [0,1]
f3 عدد المستخدمين المختلفين لنفس المصدر count
f4 أقصى اندفاع محاولات داخل sub-window قصيرة count
f5 متوسط الفاصل الزمني بين المحاولات seconds
f6 انحراف الفاصل الزمني seconds
f7 وزن زمني مقارنةً بخط أساس تاريخي ratio [0,1]
f8 علامة whitelist 0 أو 1

6) طريقة الاستخدام

6.1 تثبيت المتطلبات

pip install -r requirements.txt

6.2 تدريب النموذج (إذا احتجت إعادة توليد artifacts)

python train_intrusion_model.py --n-samples 120000 --attack-ratio 0.08 --label-noise 0.01 --seed 42 --save-dataset

6.3 تشغيل الخدمة

python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload

7) المدخلات (Input)

المسار المستخدم: POST /wazuh/event

الحقل النوع الوصف
timestamp string وقت الحدث بصيغة ISO.
rule object بيانات قاعدة التنبيه.
agent object بيانات العميل/الوكيل.
data.srcip string عنوان المصدر.
data.srcuser string اسم المستخدم.
data.status string failed أو success.
full_log string النص الخام للسجل.

7.1 مثال Input

{
  "timestamp": "2026-05-01T10:00:00Z",
  "rule": {"id": "5710", "groups": ["syslog", "sshd"]},
  "agent": {"id": "001", "name": "target-host"},
  "data": {"srcip": "10.20.30.40", "srcuser": "root", "status": "failed"},
  "full_log": "Failed password for root from 10.20.30.40"
}

8) المخرجات (Output)

الحقل التفسير
status حالة التنفيذ (processed).
src_ip عنوان المصدر بعد المعالجة.
features متجه الميزات f1..f8.
risk_score درجة الخطورة الناتجة عن النموذج.
threshold عتبة القرار المعتمدة.
predicted_label 0 طبيعي، 1 مشبوه.
decision allow أو block.
is_whitelisted نتيجة فحص whitelist.
drift_monitor مؤشرات انحراف البيانات.

8.1 مثال Output

{
  "status": "processed",
  "src_ip": "10.20.30.40",
  "features": [6.0, 0.66, 3.0, 3.0, 18.0, 15.67, 0.61, 0.0],
  "risk_score": 0.73,
  "threshold": 0.5293,
  "predicted_label": 1,
  "decision": "block",
  "is_whitelisted": false,
  "drift_monitor": {"enabled": true, "ready": true, "drift_level": "low"}
}

9) نقاط مهمة لفهم السلوك

f7 قد تبدأ بـ 0.5

عند قلة التاريخ الزمني تكون القيمة حيادية، ثم تصبح ديناميكية مع تراكم البيانات.

drift_monitor.ready

تبقى false في البداية حتى يجمع النظام عددًا كافيًا من العينات.

10) اختبار سريع

$body = @{
  timestamp="2026-05-01T10:00:00Z"
  rule=@{id="5710"; groups=@("syslog","sshd")}
  agent=@{id="001"; name="target-host"}
  data=@{srcip="10.20.30.40"; srcuser="root"; status="failed"}
  full_log="Failed password"
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Uri "http://127.0.0.1:8000/wazuh/event" -Method Post -ContentType "application/json" -Body $body

11) المشاكل الشائعة

المشكلة السبب الحل
Model artifacts not loaded ملفات artifacts غير موجودة شغّل التدريب أو تأكد من مسار الملفات.
422 Validation Error صيغة payload غير صحيحة التزم ببنية الإدخال في القسم 7.
قرار غير متوقع عتبة القرار/توزيع البيانات راجع threshold وأعد التدريب عند الحاجة.
تم إعداد هذا التقرير لتقديم شرح واضح وكامل للمشروع من منظور تقني وتشغيلي.
